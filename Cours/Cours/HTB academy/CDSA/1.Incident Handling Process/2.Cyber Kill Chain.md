# Cyber Kill Chain
![Cyber_kill_chain.png](Cyber_kill_chain.png)
1. Reconnaissance (Recon stage)
L'attaquant choisit sa cible et commence à récolter des informations.
Méthodes passives : réseaux sociaux (LinkedIn, Instagram), sites web de l'entreprise, offres d'emploi (qui révèlent souvent les outils utilisés comme antivirus, OS, technologies réseau...).
Méthodes actives : scans des applications web et adresses IP de la cible.
2. Armement (Weaponize stage)
L'attaquant prépare un logiciel malveillant (malware) léger et discret, difficile à détecter par antivirus/EDR.
Ce malware est conçu pour obtenir un accès à distance persistant, et pour pouvoir installer d’autres outils plus tard.
3. Livraison (Delivery stage)
Le malware ou l'exploit est envoyé à la victime.
Méthodes :
```
Emails de phishing avec pièce jointe ou lien piégé.
Pages web piégées qui ressemblent aux vrais sites de l’entreprise (pour voler des identifiants ou faire télécharger le malware).
Appels téléphoniques de manipulation (social engineering) pour convaincre la cible d’exécuter un fichier.
```
Souvent, il suffit d’un simple double-clic sur un fichier (.exe, .bat, .vbs, .js, .hta…).
Parfois, le malware est déposé via une clé USB laissée intentionnellement.
4. Exploitation (Exploitation stage)
C’est le moment où l’exploit ou le malware est activé.
L’attaquant exécute du code sur la machine cible pour obtenir un accès ou un contrôle.
1. Lors de la phase d’`installation`, le **stager** initial est exécuté et fonctionne sur la machine compromise. Comme mentionné précédemment, la phase d’installation peut être réalisée de différentes manières, en fonction des objectifs de l’attaquant et de la nature de la compromission. Voici quelques techniques couramment utilisées pendant la phase d’**installation** :
- **Droppers** : Les attaquants peuvent utiliser des **droppers** pour déposer du **malware** sur le système cible. Un **dropper** est un petit morceau de code conçu pour installer un **malware** sur le système et l’exécuter. Le **dropper** peut être diffusé via différents vecteurs, comme des pièces jointes d’emails, des sites web malveillants ou des techniques d’**engineering social**.
- **Backdoors** : Une **backdoor** est un type de **malware** conçu pour fournir à l’attaquant un accès persistant au système compromis. La **backdoor** peut être installée pendant la phase d’exploitation (exploitation stage) ou être déposée via un **dropper**. Une fois installée, elle peut être utilisée pour mener d'autres attaques ou voler des données sur le système compromis.
- **Rootkits** : Un **rootkit** est un type de **malware** conçu pour masquer sa présence sur un système compromis. Les **rootkits** sont souvent utilisés durant la phase d’**installation** afin d’échapper à la détection par les antivirus ou autres outils de sécurité. Ils peuvent être installés lors de la phase d’exploitation ou également livrés via un **dropper**.
1. Command and Control (C2 stage)
L’attaquant établit une connexion d'accès à distance sur la machine compromise.
Souvent, il utilise un stager modulaire qui peut charger d'autres scripts à la volée ("on-the-fly").
Les groupes avancés déploient plusieurs versions différentes de leur malware dans le réseau.
👉 Cela garantit que si un malware est détecté et neutralisé, ils peuvent revenir grâce à une autre version encore active.
2. Actions on Objective
C’est la phase où l'attaquant réalise son objectif final.
Objectifs possibles :
```
Exfiltration de données sensibles.
Escalade de privilèges maximale dans le réseau.
Déploiement de ransomware (le ransomware chiffre ou rend les données inaccessibles, et exige une rançon pour les récupérer — paiement fortement déconseillé).
```
🧠 Note importante :
```
Les attaquants ne suivent pas toujours un chemin linéaire (comme présenté dans le modèle de la cyber kill chain).
Certaines étapes (comme installation ou recon) peuvent être répétées plusieurs fois pour étendre l'accès, trouver de nouvelles cibles internes, ou affiner leur attaque avant d'atteindre l'objectif final.
```
