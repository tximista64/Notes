# Incident Handling Definition & Scope
# **Définition et périmètre de l'Incident Handling**
Un `event` (événement) est une action se produisant dans un système ou un réseau. Exemples d’**events** :
- Un utilisateur qui envoie un email
- Un clic de souris
- Un firewall (pare-feu) qui autorise une requête de connexion
Un `incident` est un **event** ayant une conséquence négative.
Un exemple d’**incident** est un crash système. Un autre exemple est un accès non autorisé à des données sensibles. Les **incidents** peuvent aussi être causés par des catastrophes naturelles, des coupures de courant, etc.
Il n’existe pas de définition unique d’un **IT security incident**, cela varie donc d’une organisation à l’autre.
Nous définissons un **IT security incident** comme un **event** dont l’intention est clairement de nuire, et qui est dirigé contre un système informatique. Exemples d’**incidents** :
- Vol de données
- Vol de fonds
- Accès non autorisé à des données
- Installation et utilisation de malware (logiciel malveillant) et d’outils de remote access (accès à distance)
L'`Incident handling` est un ensemble de procédures clairement définies pour **gérer et répondre aux incidents de sécurité** dans un environnement informatique ou réseau.
Une des ressources les plus utilisées sur l’**incident handling** est le [**Computer Security Incident Handling Guide du NIST**](https://nvlpubs.nist.gov/nistpubs/SpecialPublications/NIST.SP.800-61r2.pdf).
