## **DMARC**
[DMARC](https://dmarcly.com/blog/how-to-implement-dmarc-dkim-spf-to-stop-email-spoofing-phishing-the-definitive-guide#what-is-dmarc) est une protection contre le phishing basée sur les mécanismes existants [SPF](https://dmarcly.com/blog/how-to-implement-dmarc-dkim-spf-to-stop-email-spoofing-phishing-the-definitive-guide#what-is-spf) et [DKIM](https://dmarcly.com/blog/how-to-implement-dmarc-dkim-spf-to-stop-email-spoofing-phishing-the-definitive-guide#what-is-dkim).
L’objectif est de **rejeter les emails qui prétendent provenir de votre organisation** (ex. demande de paiement frauduleuse d’un soi-disant employé).
DMARC est **simple et peu coûteux à mettre en place**, mais **des tests approfondis sont indispensables**, sous peine de **bloquer des emails légitimes sans possibilité de les récupérer**.
Certains systèmes d’emails ajoutent un **en-tête** indiquant le résultat du test DMARC. Cela permet, avec des règles de filtrage, d’aller plus loin en bloquant ou signalant les emails échouant le test DMARC, même s’ils ne proviennent pas de vos domaines. Attention : risque élevé de faux positifs, notamment avec des services tiers envoyant des emails “on behalf of”.

### **Renforcement des Endpoints (et EDR)**
- **Désactiver LLMNR / NetBIOS**
📺 [Voir vidéo](https://youtu.be/uFiFjHvIaMI)
- **Implémenter LAPS** et retirer les privilèges admin des utilisateurs standards
📺 [Voir vidéo LAPS](https://youtu.be/06-QiwKDFEo)
- **Désactiver ou restreindre PowerShell** en mode `ConstrainedLanguage`
- **Activer les règles ASR** (Attack Surface Reduction) de Microsoft Defender
📺 [ASR et Exploit Protection](https://youtu.be/gCcWKtyFJBU)
- **Implémenter du whitelisting** : au minimum, **bloquer l’exécution depuis les dossiers utilisateur** (Downloads, Desktop, AppData…).
Bloquer aussi les scripts `.hta`, `.vbs`, `.cmd`, `.bat`, `.js`, etc.
⚠️ Ne pas oublier les [**LOLBin**](https://lolbas-project.github.io/) (Living Off The Land Binaries) souvent utilisés pour contourner les protections.
- **Utiliser un pare-feu local** : bloquer la communication poste-à-poste et les connexions sortantes vers les LOLBins.
- **Déployer un EDR** avec intégration **AMSI** pour inspection des scripts obfusqués.
📺 [Présentation AMSI](https://youtu.be/wBK1fTg6xuU)

### **Protection Réseau**
- **Segmentation réseau impérative**
- Aucune ressource interne ne doit être exposée directement à Internet (hors DMZ)
- **IDS/IPS** avec inspection SSL/TLS fortement recommandée pour analyser le contenu réel du trafic
- **Accès réseau restreint** aux seuls équipements autorisés : utiliser **802.1x**
- Pour les environnements cloud-only (Azure/AD), utiliser les **Conditional Access Policies** (accès uniquement depuis un appareil géré par l’entreprise)
---
### **Gestion des identités / MFA / Mots de passe**
- Le **vol de credentials** privilégiés est la méthode la plus courante d'escalade dans les environnements Active Directory
- Éviter les mots de passe **complexes mais faibles**
- **MFA obligatoire** pour tout accès administratif à n’importe quelle application ou équipement
