# Structure of InfoSec
Ce module offre une introduction structurée au monde de la sécurité de l'information (InfoSec), avec une vision globale des rôles, des domaines d'expertise, et des perspectives professionnelles. Il est destiné aux débutants souhaitant s’orienter dans l’univers de la cybersécurité.
## Objectif
Donner une vue d'ensemble de l'organisation de l'InfoSec afin d'aider le lecteur à choisir une direction de spécialisation future.
> **Note de l’auteur :**
> Ce module est volontairement théorique, car commencer sans les bases reviendrait à tenter de piloter un avion de chasse sans connaître les commandes.
## Pourquoi l'InfoSec est-elle essentielle ?
Dans un monde où la plupart de nos activités sont numériques (communication, banque, e-commerce, business), il est vital de protéger les données contre les accès non autorisés.
L’**Information Security** consiste à empêcher :
- la visualisation non autorisée,
- la modification,
- ou la destruction de données.
## Panorama simplifié de l’environnement numérique

- **Client** : PC/ordinateur portable utilisé pour accéder à Internet.
- **Internet** : Réseau mondial de serveurs offrant des services comme Hack The Box.
- **Servers** : Hébergent des services comme les web servers.
- **Network** : Ensemble d’ordinateurs/serveurs interconnectés.
- **Cloud** : Data centers avec serveurs mutualisés.
- **Blue Team** : Défend les actifs de l’entreprise.
- **Red Team** : Simule une attaque pour tester les défenses.
- **Purple Team** : Synergie Blue + Red pour l’amélioration continue.
## Métaphore du château
- **Trésor** : Vos données.
- **Murs** : Firewalls, chiffrement.
- **Gardes** : Protocoles de sécurité, contrôle d’accès.
- **Knights** (Penetration Testers) : Testent les défenses.
- **Digital Transformation** : Expansion du château → cible plus attrayante.
- **Cyber Threats** : Les assaillants.

## Concepts de sécurité
- **Risk** : Combinaison de la probabilité et de l’impact potentiel.
- **Threat** : Ce qui peut causer un incident (acteur ou événement).
- **Vulnerability** : Faiblesse exploitable dans un système.
Les trois sont liés :
> Risk = Threat × Vulnerability × Impact
## Rôles en InfoSec
| Rôle                                      | Description                                            | Lien avec le pentesting                            |
| ----------------------------------------- | ------------------------------------------------------ | -------------------------------------------------- |
| Chief Information Security Officer (CISO) | Supervise la stratégie globale de sécurité             | Définit le cadre des tests de sécurité             |
| Security Architect                        | Conçoit les systèmes sécurisés                         | Cible des attaques simulées                        |
| Penetration Tester                        | Réalise des attaques éthiques pour trouver des failles | Rôle cible principal                               |
| Incident Response Specialist              | Répond aux incidents et collabore avec les pentesters  | Analyse post-attaque, coordination                 |
| Security Analyst                          | Surveille les menaces et données de sécurité           | Exploite les rapports de test                      |
| Compliance Specialist                     | Vérifie le respect des normes et règlements            | Utilise les rapports de pentest pour la conformité |
Ce panorama constitue une fondation indispensable pour toute personne souhaitant évoluer dans l’univers de la cybersécurité.
